一、项目概述
1.1 基本信息
项目名	WoT Mods Manager
描述	坦克世界（World of Tanks）模组管理桌面客户端
技术栈	Rust + Tauri (v2) + React + TypeScript
目标平台	Windows 10/11
许可证	MIT / GPL-3.0
1.2 项目目标
为《坦克世界》玩家提供一个轻量、安全、易用的模组管理工具，解决以下痛点：

模组文件散落、管理混乱

手动备份/恢复繁琐

模组版本与游戏版本兼容性不明

多模组同名文件冲突难以处理

二、核心功能需求
2.1 模组管理
2.1.1 模组列表
以卡片/列表视图展示所有已安装模组

支持按名称、状态、分类、安装时间排序和筛选

显示模组：名称、版本、作者、状态（启用/禁用/过时/冲突）、安装时间、文件大小

2.1.2 模组安装
本地安装（主要方式）

支持拖拽 .zip、.wotmod 文件到应用窗口安装

支持通过文件选择对话框选择安装包

支持批量安装多个模组

内置模组包

软件安装包内预置常用模组集合（如 XVM、瞄准插件等）

用户可选择从内置包一键安装

2.1.3 模组操作
启用/禁用（切换激活状态）

卸载（安全删除，含文件清理）

批量启用/禁用/卸载

查看模组详情（元数据、文件结构、依赖关系）

2.1.4 更新检测
检查已安装模组是否有新版本（基于本地缓存或用户手动指定版本信息）

2.2 冲突处理（核心特性）
问题背景：不同模组解压后可能包含同名文件夹（如 res_mods/1.18.0.1/gui/），直接覆盖会导致模组互相破坏。

解决方案：

安装时冲突检测

解压前扫描目标目录，检测是否存在同名文件/文件夹

如有冲突，展示冲突文件列表供用户选择处理方式

冲突处理策略

智能合并：自动合并不同模组的文件，同名文件按优先级保留

覆盖：用新模组文件覆盖旧文件（警告用户）

跳过：保留现有文件，不安装冲突部分

重命名：将新文件重命名为 filename_conflict_xxx.ext

创建隔离目录：为每个模组创建独立子目录，通过加载器配置管理

冲突标记

在模组列表中标记存在冲突的模组

显示冲突详情（与哪些模组冲突、冲突文件列表）

2.3 游戏集成
自动检测：扫描注册表（HKEY_LOCAL_MACHINE\SOFTWARE\Wargaming.net\WOT）和常见安装目录（C:\Games\World_of_Tanks、C:\Program Files\World_of_Tanks）

手动配置：允许用户手动指定游戏安装路径

版本检测：读取游戏 version.xml 或 game_version 文件获取版本号

一键启动：从管理器直接启动游戏（支持启动参数配置）

多区服支持：支持 EU/NA/ASIA/RU/CN 区服切换

2.4 备份与恢复
安装前自动备份：安装新模组前自动备份将被修改的文件

手动备份：用户可创建当前模组配置快照

备份管理：查看、删除、恢复备份

备份内容：模组文件 + 模组列表状态

2.5 配置管理
配置文件编辑：直接在应用内编辑模组配置文件（如 config.xml、settings.json）

配置预设：保存和加载常用模组配置方案

配置导入/导出：分享配置方案（JSON 格式）

2.6 系统集成
单实例：确保应用只运行一个实例

文件关联：关联 .wotmod 文件格式，双击直接打开

系统托盘：快速启用/禁用模组、启动游戏、退出应用

开机启动（可选）：开机自动启动，托盘常驻

4.2 冲突检测算法
扫描目标目录：递归扫描模组将要安装到的目标目录（如 res_mods/1.18.0.1/）

构建已安装文件索引：记录所有已安装文件的相对路径和哈希值

对比新模组文件：遍历新模组解压后的所有文件

冲突判定：

如果文件路径已存在 → 冲突

如果文件路径不存在 → 无冲突

冲突分级：

高冲突：同名且内容不同（哈希不同）

中冲突：同名但来自相同模组的不同版本

低冲突：同名且内容相同（可安全覆盖）

4.3 冲突处理策略实现
rust
pub enum ConflictStrategy {
    /// 智能合并 - 按优先级保留文件
    SmartMerge,
    /// 覆盖 - 新文件覆盖旧文件
    Overwrite,
    /// 跳过 - 保留旧文件
    Skip,
    /// 重命名 - 新文件重命名后安装
    Rename,
    /// 隔离 - 安装在独立子目录
    Isolate,
}
4.4 模组启用/禁用机制
方式一：文件重命名（推荐）

启用：将 .wotmod 保持原样

禁用：将 .wotmod 重命名为 .wotmod.disabled

方式二：文件移动

将模组文件移出加载目录（如移到 _disabled/ 子目录）

方式三：游戏加载器配置

部分模组支持通过配置文件启用/禁用

本方案采用方式一 + 方式二组合：主要使用文件重命名，对于不支持重命名的模组使用移动方式。

游戏路径自动检测（Windows 注册表）
rust
use winreg::RegKey;
use winreg::enums::HKEY_LOCAL_MACHINE;

pub fn detect_wot_path() -> Option<PathBuf> {
    let hklm = RegKey::predef(HKEY_LOCAL_MACHINE);
    let path = r"SOFTWARE\Wargaming.net\WOT";
    if let Ok(key) = hklm.open_subkey(path) {
        if let Ok(install_path) = key.get_value::<String, _>("InstallDir") {
            return Some(PathBuf::from(install_path));
        }
    }
    // 常见安装目录回退
    let common_paths = vec![
        "C:\\Games\\World_of_Tanks",
        "C:\\Program Files\\World_of_Tanks",
        "C:\\Program Files (x86)\\World_of_Tanks",
    ];
    for path in common_paths {
        let p = PathBuf::from(path);
        if p.exists() {
            return Some(p);
        }
    }
    None
}

6.4 冲突处理关键代码结构
rust
// core/conflict.rs
pub struct ConflictDetector;

impl ConflictDetector {
    /// 检测模组安装时的文件冲突
    pub fn detect_conflicts(
        game_dir: &Path,
        mod_files: &[ModFile],
        installed_mods: &[ModInfo],
    ) -> Vec<ConflictInfo> {
        let mut conflicts = Vec::new();
        for file in mod_files {
            let target_path = game_dir.join(&file.relative_path);
            if target_path.exists() {
                // 计算已存在文件的哈希
                let existing_hash = hash_file(&target_path);
                if existing_hash != file.hash {
                    conflicts.push(ConflictInfo {
                        file_path: file.relative_path.clone(),
                        conflict_with: find_owners(&target_path, installed_mods),
                        resolution: ConflictResolution::Pending,
                    });
                }
            }
        }
        conflicts
    }
}

十、风险与应对
风险	应对措施
游戏更新导致模组路径变化	定期更新游戏版本兼容列表，支持用户手动配置路径
模组文件损坏或不完整	安装前验证 ZIP 完整性，计算文件哈希
权限不足（Program Files 写入）	提示用户以管理员身份运行，或建议将游戏安装在非系统目录
大模组包解压慢	使用异步流式解压，显示进度条
同名文件冲突导致游戏崩溃	完善的冲突检测机制，安装前预览冲突并让用户选择处理方式
